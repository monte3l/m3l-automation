### Verified and discarded (for the record)

- **SF-2 (WS-6, dropped 2026-07-06)** ŌĆö the "lying-low ZIP entry spikes peak memory" premise does **not** reproduce against the pinned `adm-zip@0.5.18`. `getData()` self-bounds decompression to the **declared** `header.size` (`zipEntry.js`: `Buffer.alloc(_centralHeader.size)` + `inflateRawSync(..., { maxOutputLength })`) and throws `BAD_CRC` on a mismatch ŌĆö so a lying-low entry either throws or returns Ōēż declared bytes, never more. Since the declared size is already gated against the budget (`zip.ts:231`), the proposed post-`getData()` recheck (`data.length > remainingBytes`) can never trip and would defend an unreachable path. The existing declared-size gate plus the `maxTotalBytes`/`maxEntries`/depth caps are sufficient; no code change shipped.
- Provenance "orphaned commit" claim ŌĆö false (`b40458bb` is contained in `main`; `check:provenance` green).
- importers/exporters "missing `expectTypeOf`" ŌĆö false (17 and 34 usages).
- 14 static AWS SDK imports ŌĆö documented ADR-0017 first-class exception.
- `CHANGELOG.md` absence ŌĆö semantic-release generates it on first publish.
- Nits (excluded per mandate): `.worktreeinclude` absent `.env` warnings, redact single-quote tail (documented best-effort), `M3LPollDecision` failure-arm payload, `defaultHeaders` input mutability, `M3LHttpRequestEvent.method` literal, badge cosmetics.
