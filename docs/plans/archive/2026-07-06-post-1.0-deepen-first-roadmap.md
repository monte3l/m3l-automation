# Post-1.0.0 roadmap — deepen-first 1.1 wave, then the consumer fleet

- **Date:** 2026-07-06 · **Baseline:** `main` at 1.0.0 (22/22 submodules, all
  gates green)
- **Authority:** implements the direction recorded in
  [ADR-0021](../adr/0021-post-1.0-deepen-first-strategy.md) (deepen first,
  consume later). Builds on the pre-1.0.0 audit plan
  (`docs/plans/2026-07-05-pre-1.0.0-release-audit.md`) — referenced, never
  edited.
- **Operating model:** every code phase runs the hub-and-spoke pipeline:
  `/start-work` decision gate → branch off `main` → doc-first contract update
  → `test-author` (RED) → `submodule-implementer` (GREEN) → reviewer fan-out
  (`code-reviewer` + `spec-conformance-reviewer`, plus `type-design-analyzer`
  for new public types and `silent-failure-hunter` for error/async surfaces)
  → `/syncing-docs` → Conventional Commit → PR with `claude-pr-review` PASS.
  The hub updates `docs/implementation-status.md` notes after each phase.
  Fixed per-feature doc overhead (~30 min): reference-page update, provenance
  re-stamp, `check:doc-exports`, `gen:index` + `check:index`, `lint:md`.

## Phase 0 — design gate (maintainer, no code)

Two decisions block later phases; settle them first.

1. **Preset-inheritance merge semantics** (blocks Phase 3b): shallow merge
   (derived key fully overrides base key) vs deep merge; array
   replace-vs-concat. _Feasibility recommendation: shallow merge first,
   documented explicitly; revisit deep merge only on demand._
2. **Dependency-layering rules for ESLint zones** (blocks Phase 4): which
   cross-submodule imports are forbidden (e.g. nothing imports `core/script`;
   `aws/*` imports only `core/errors` + type-only surfaces — codify the
   current actual graph). ADR-0009 is already Accepted; this is execution.

## Phase 1 — quick wins (parallelizable, independent)

### WS-A — registerable text extractors — `feat/text-extractor-registration`

- **What:** surface the existing registration seam as a documented public
  extension point: re-export the `M3LTextExtractor` contract through the core
  barrel path where missing, document `M3LTextExtractorRegistry.register()`
  and constructor-injected extractor precedence, add a custom-extractor
  example.
- **Seam:** `packages/m3l-common/src/core/text/registry.ts` (`register()`),
  `src/core/text/contract.ts` (`M3LTextExtractor`).
- **Docs:** `docs/reference/core/text.md` extensibility section.
- **Semver:** additive-minor. **Effort:** S.

### WS-B — coverage backfill — `test/coverage-backfill`

- **What:** close the known soft spots — `M3LPoller.ts` lines 117–118
  (attempt-counter edge), `M3LPrompt.ts` branch gaps (input-validation error
  paths), email extractor branches.
- **Semver:** none (tests only). **Effort:** S.
- **Gate:** `test:coverage` per-file thresholds; read
  `coverage-final.json`, not the text table (work-log lesson).

## Phase 2 — observability (parallelizable in worktrees, ADR-0013)

### WS-C — polling/retry telemetry — `feat/polling-telemetry`

- **What:** optional observability wiring on `M3LPoller` and
  `M3LRetryRunner`: an opt-in emitter/logger constructor option and a typed
  event map (attempt, success, failure, exhausted — attempt number, backoff
  delay, classified error summary). Neither class emits anything today.
- **Seams:** `src/core/polling/M3LPoller.ts`, `M3LRetryRunner.ts`; reuse
  `core/events` `M3LEventEmitterBase` and/or `core/logging` ports.
- **Docs:** `docs/reference/core/polling.md` event-map section.
- **Semver:** additive-minor (optional options, new exported event types).
  **Effort:** M.

### WS-D — correlation-ID tracing — `feat/script-correlation-id`

- **What:** optional per-run correlation ID: seeded via `M3LScriptOptions`
  (generated when absent), exposed on hook contexts, auto-threaded into
  `M3LLogEvent` data and `M3LError` context for errors raised through script
  stages.
- **Seams:** `src/core/script/M3LScript.ts` + `M3LScriptOptions.ts` (hook
  context), `src/core/logging/M3LLogEvent.ts`, `src/core/errors/M3LError.ts`
  (existing optional `context`).
- **Docs:** `docs/reference/core/script.md` + `core/logging.md`.
- **Semver:** additive-minor (optional fields only). **Effort:** M.
- **Review note:** include `security-reviewer` — the ID flows through the
  redaction path and stderr diagnostics.

## Phase 3 — config & preset ergonomics

### WS-E — schema-time config validators — `feat/config-validators`

- **What:** optional `validator` on `M3LConfigParameterOptions`, executed
  after coercion with failures surfaced through the existing config error
  channel; stock helpers (`range`, `regex`, `predicate`) exported through the
  core barrel.
- **Seams:** `src/core/config/M3LConfigParameter.ts`,
  `coerceConfigValue.ts`.
- **Docs:** `docs/reference/core/config.md`.
- **Semver:** additive-minor. **Effort:** M.

### WS-F — preset inheritance — `feat/preset-inheritance`

- **Depends on:** Phase 0 decision 1 (merge semantics).
- **What:** optional top-level `extends` in presets; recursive base load with
  the decided merge semantics; cycle detection with a new error code
  (`ERR_PRESET_CYCLE`) registered in `M3L_ERROR_CODES` — the source-scan
  completeness guard must stay green; respect the existing depth guard.
- **Seams:** `src/core/script/M3LScriptPresetLoader.ts`,
  `src/core/errors` code registry.
- **Docs:** `docs/reference/core/script.md` preset section.
- **Semver:** additive-minor. **Effort:** M.

## Version checkpoint — 1.1.0

When WS-A through WS-F are merged and all CI gates are green on `main`:
hand-bump `packages/m3l-common/package.json` to **1.1.0** (manual versioning
per ADR-0020) in a `chore(m3l-common): bump version to 1.1.0` commit. The
`exports` map is untouched throughout; `check:api` green at every merge is
the proof.

## Phase 4 — tooling residual (any time after Phase 0 decision 2)

### WS-G — ESLint dependency zones — `chore/eslint-dependency-zones`

- **What:** wire `import-x/no-restricted-paths` zones per the decided
  layering rules (ADR-0009 execution). Repo tooling only — no runtime or

- **Files:** `eslint.config.js`. **Semver:** none. **Effort:** S–M.

## Phase 5 — consumer fleet (D1, the next iteration after 1.1.0)

Per ADR-0021, this follows the wave immediately; it is scoped here so the
sequence is durable, and detailed planning happens in its own plan document
when it starts.

- Restore the `scripts/*` workspace: `pnpm-workspace.yaml`, root
  `tsconfig.json` reference, `knip.json` workspace block, `eslint.config.js`
  scope (the removal commit `3ded259f` shows the exact inverse diff), and
  extend the `guard-branch-isolation.mjs` protected-path regex to cover
  `scripts/*/src/`. CI, vitest, and turbo are pattern-based and need ~no
  changes.
- Add **real, value-shipping automation scripts** (consistent with ADR-0019,
  which removed a hollow example): consumption via `workspace:*`; MONOREPO
  mode anchors shared `data/` at the repo root. `knip` fails hollow scripts —
  that is the intended forcing function.
- Known caveat to design around: `better-sqlite3` is a native hard dep —
  Lambda-deployed scripts need arch-matched builds or must avoid the FTS
  feature.
- From this phase onward, observed usage — not speculation — feeds the
  library backlog.

## Gated follow-ons (recorded, deliberately unscheduled)

- **Broaden (D4)** — each requires a named consumer call-site before
  scaffolding: SSM config provider (`M3LConfigProvider` subclass over the
  existing `provider.ssm`, no new deps), SES messaging transport (new
  optional caret-ranged peer `@aws-sdk/client-sesv2` per ADR-0017,
  lazy-loaded, `aws` namespace), Lambda invoke wrapper (no new deps).
  The Slack/webhook transport is additionally **blocked** on `M3LHttpClient`
  gaining POST support — schedule that enhancement first if a webhook
  consumer appears.
- **Platform extraction (D5)** — minimal path (copy + parameterize the
  agent/skill/hook/bin system into one host, ~1 day) becomes actionable when
  a second repo exists; the portability census (14% agnostic / 53%
  parameterizable / 33% coupled) is in the review record. Do not productize
  before one successful port.
